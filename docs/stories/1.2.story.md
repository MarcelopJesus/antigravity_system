## Story

**Como** administrador do sistema de artigos SEO,
**Quero** que todas as imagens geradas e publicadas tenham alt text descritivo com a keyword,
**Para que** o SEO de imagens seja otimizado e nao haja penalidade por imagens sem alt text.

---

## Problema

O Visual Agent gera imagens e faz upload via WordPress REST API, mas nao seta alt text programaticamente. Cada `<img>` no artigo fica sem atributo `alt` — penalidade direta no SEO.

---

## Acceptance Criteria

- [ ] AC1: Toda imagem uploadada via Visual Agent tem alt text setado via WP REST API
- [ ] AC2: Alt text contem a keyword principal + descricao contextual
- [ ] AC3: Alt text tem no maximo 125 caracteres
- [ ] AC4: Todas as `<img>` tags no HTML do artigo publicado possuem atributo `alt` preenchido
- [ ] AC5: Testes existentes continuam passando (zero regressao)

---

## Dev Notes

### Abordagem Tecnica

- WordPress REST API para media: `POST /wp-json/wp/v2/media/{id}` aceita campo `alt_text`
- No upload da imagem, setar `alt_text` no request
- Gerar alt text: combinar keyword + contexto da secao onde a imagem aparece
- Formato: `{keyword} - {descricao contextual}` (max 125 chars)

### Arquivos Afetados
- `core/agents/visual.py` — Gerar alt text descritivo por imagem
- `core/wordpress_client.py` — Enviar alt_text no upload de media

### Regra de Negocio
- Alt text = keyword principal + descricao contextual (max 125 chars)
- Se imagem e featured image, alt text = titulo do artigo
- Se imagem e de secao, alt text = keyword + resumo do contexto da secao
