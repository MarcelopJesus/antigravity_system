## Story

**Como** administrador do sistema de artigos SEO,
**Quero** que o schema markup JSON-LD (Article, LocalBusiness, FAQ) seja efetivamente publicado no WordPress,
**Para que** o Google consiga interpretar os dados estruturados e exibir rich results nos resultados de busca.

---

## Problema

`inject_schema_into_html()` em `core/seo/schema.py` e um no-op. O WordPress strip `<script>` tags do conteudo enviado via REST API. O schema JSON-LD gerado (Article, LocalBusiness, FAQ) NUNCA e publicado no site final.

## Dev Notes

### Abordagem Tecnica

O WordPress REST API permite enviar custom fields junto com o post. Existem duas abordagens viaveis:

**Opcao A — Yoast SEO REST API:**
- Yoast tem endpoint para schema: `wpseo_schema` ou campo `yoast_head_json`
- Verificar se Yoast permite injetar schema customizado via REST API
- Endpoint: `POST /wp-json/wp/v2/posts/{id}` com meta fields do Yoast

**Opcao B — Custom Fields (ACF ou meta nativo):**
- Enviar schema como custom field `_seo_schema_json`
- Requer snippet PHP no tema ou plugin para renderizar o campo como `<script>` no `<head>`
- Mais controle, menos dependencia do Yoast

**Opcao C — Post meta via REST API:**
- Usar `meta` field do WordPress REST API para enviar o schema
- Criar um mu-plugin simples no WordPress que le o meta e injeta no head

### Arquivos Afetados
- `core/seo/schema.py` — Gerador de schema (ja funciona, problema e na injecao)
- `core/wordpress_client.py` — Publicacao via REST API (precisa enviar schema)
- `core/pipeline.py` — Pode precisar ajuste na ordem dos steps
