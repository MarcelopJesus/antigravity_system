# Story 1.4 — IMG_PLACEHOLDER restante no HTML publicado

## Story

**Como** administrador do sistema de artigos SEO,
**Quero** que nenhum `<!-- IMG_PLACEHOLDER -->` permaneca no HTML do artigo publicado,
**Para que** o codigo HTML seja limpo e profissional, sem lixo de processamento.

---

## Problema

Se a geracao de imagem falha (timeout, API error, etc.), o placeholder `<!-- IMG_PLACEHOLDER -->` permanece no HTML publicado. Nao e visivel ao usuario, mas e lixo no codigo-fonte.

---

## Acceptance Criteria

- [ ] AC1: HTML publicado no WordPress nunca contem `<!-- IMG_PLACEHOLDER -->`
- [ ] AC2: Se imagem falha, placeholder e removido do HTML antes de publicar
- [ ] AC3: Pipeline trata falha de imagem gracefully (log warning, remove placeholder)
- [ ] AC4: Dry-run tambem limpa placeholders do HTML em output/
- [ ] AC5: Testes existentes continuam passando (zero regressao)

---

## Dev Notes

### Abordagem Tecnica

- Adicionar step de cleanup no pipeline ANTES da publicacao
- Regex simples: `re.sub(r'<!--\s*IMG_PLACEHOLDER\s*-->', '', html)`
- Pode ser feito no `pipeline.py` (step final) ou no `editor.py` (polish final)
- Tambem limpar variantes: `[IMG_PLACEHOLDER]`, `{IMG_PLACEHOLDER}`

### Arquivos Afetados
- `core/pipeline.py` — Adicionar step de limpeza pre-publicacao
- `core/agents/editor.py` — Alternativa: limpar no polish

### Decisao
- Preferir `pipeline.py` (step final, catch-all) para garantir que NENHUM placeholder escape
