## Story

**Como** administrador do sistema de artigos SEO,
**Quero** que meta descriptions fora do range ideal nao resultem em pontuacao zero no SEO Scorer,
**Para que** artigos com meta descriptions ligeiramente acima do limite nao sejam penalizados desproporcionalmente.

---

## Problema

SEO Scorer da 0 pontos se meta_description tem 156 chars (1 char acima de 155). Artigo `ansiedade-generalizada` perdeu 10 pontos por 1 caractere. O scoring e binario (tudo ou nada) quando deveria ter pontuacao parcial.

---

## Acceptance Criteria

- [ ] AC1: Meta description com 120-155 chars recebe pontuacao maxima (10 pts)
- [ ] AC2: Meta description com 156-165 chars recebe pontuacao parcial (5 pts)
- [ ] AC3: Meta description acima de 165 chars ou abaixo de 70 chars recebe 0 pts
- [ ] AC4: Prompt do Analyst enforca limite de 150 chars (margem de seguranca)
- [ ] AC5: Testes existentes atualizados para novo scoring
- [ ] AC6: Zero regressao nos demais testes

---

## Dev Notes

### Abordagem Tecnica

**Duas mudancas complementares:**

1. **SEO Scorer — Pontuacao parcial:**
   - 120-155 chars → 10 pts (ideal)
   - 100-119 chars → 7 pts (aceitavel)
   - 156-165 chars → 5 pts (marginal, mas nao zero)
   - <70 ou >165 chars → 0 pts

2. **Analyst Prompt — Margem de seguranca:**
   - Atualizar prompt para enforcar "maximo 150 caracteres" (margem de 5 chars)
   - Atual provavelmente diz "155 caracteres"

### Arquivos Afetados
- `core/agents/seo_scorer.py` — Logica de scoring de meta description
- `config/tenants/_default/prompts/analyst.txt` — Prompt do Analyst
